# 教育行业客户深度研究技能

为 WPS 365 售前团队提供专业的教育行业客户研究支持，自动识别客户类型，生成结构化分析报告和解决方案匹配。

## ⚠️ 重要执行要求

### 1. 【默认执行】历史合作信息获取（⚠️ 强制执行）

**⚠️ 警告：这是必执行任务，不可跳过**

**执行方式（两种方式必须都用）**：

#### 来源1：大模型自身知识（优先使用）
- 大模型的预训练数据中已包含大量 WPS 与高校合作的公开信息
- **必须主动调用**大模型知识，询问："北京航空航天大学和WPS/金山办公有什么合作历史？"
- 这是**首选信息源**，因为大模型知识库包含大量公开合作案例

#### 来源2：WebSearch 补充（验证/补充/确认）
- 使用 WebSearch 工具搜索以下组合：
  ```
  # 高等教育客户（必须搜索）
  搜索1：`{学校名} + WPS + 合作`
  搜索2：`{学校名} + 金山办公 + 案例`
  搜索3：`{学校名} + WPS 365 + 中标`
  搜索4：`{学校名} + 金山办公 + 中标`

  # K12/职业教育客户（必须搜索）
  搜索1：`{学校名} + WPS + 合作`
  搜索2：`{学校名} + 金山办公 + 案例`
  ```

**执行步骤**：
1. **Step 1**：先调用大模型自身知识，获取已知的合作信息
2. **Step 2**：使用 WebSearch 搜索上述关键词组合
3. **Step 3**：对比两种来源的信息，综合整理
4. **Step 4**：在执行摘要的"历史合作情况"章节中填写

**输出要求**：
- 报告中**必须有**"历史合作情况"章节（在执行摘要部分）
- 必须综合两种来源的信息，填写表格：
  - 合作产品（WPS Office / WPS 365 / 金山文档 / 金山协作等）
  - 合作范围（全校授权 / 特定院系 / 部门级 / 个人版使用）
  - 合作时间（起始年份）
  - 合作案例（具体项目名称）
  - 合作效果（如有公开数据）
- 如果确实无合作信息：写"未查询到公开的WPS合作信息"，然后提供"市场切入点分析"

**禁止**：
- ❌ 明明有合作却说没有
- ❌ 编造不存在的合作信息
- ❌ 跳过历史合作搜索步骤

### 2. 数据准确性

- 所有数据必须来自真实来源（官网、WebSearch等）
- 禁止编造 IT投入、招投标、决策人等信息
- 无法获取的数据标注"待确认"

## 客户类型自动识别

### 高等教育识别关键词

**办学层次**：大学、学院、高校、双一流、本科、专科、研究生院
**机构类型**：继续教育学院、国际教育学院、职业学院
**主管部门**：部属、省属、市属、民办

### 基础教育识别关键词

**K12阶段**：小学、初中、高中、完全中学、一贯制
**主管部门**：教育局、教委、教育委员会

### 职业教育识别关键词

**办学层次**：职业大学、职业技术学院、高等专科学校、技校
**办学类型**：高职、中职、技校、职业培训

### 教培机构识别关键词

**机构类型**：培训学校、补习学校、国际学校、民办学校

## 使用流程

### ⚠️ 重要：年份使用规则

**当前年份规则**：
- 报告生成时自动使用**当前系统年份**（如2026年）
- 搜索数据时优先搜索**最近年度数据**（如当前2026年，则搜索2025年数据）
- 报告中所有日期标注使用当前年份

**示例**：
```
当前时间：2026年2月
搜索年报：2025年年报、2025年年度报告
报告日期：2026-02-27
```

### 第二步：数据获取（按优先级）

按以下优先级获取客户信息：

⚠️ **重要提醒**：
- **禁止编造数据**：所有数据必须来自真实来源，不可推测或编造
- **无法获取时标注**：如果无法从公开渠道获取某项数据，明确标注"待确认"或"无法获取"
- **数据时效性**：优先使用最近12个月内的数据，并在报告中标注数据时间

#### 优先级 1：用户上传文件（最优先）
- 检查用户是否上传了客户资料、招标文件、内部文档
- 使用 Read 工具读取文件内容
- 提取关键信息：学校规模、业务结构、痛点、预算等

#### 优先级 2：学校官网/官方渠道
- 学校官网：了解业务介绍、组织架构、新闻动态

- 招生就业：办学规模、就业质量
- 新闻动态：数字化转型动态
- 人才招聘：IT团队规模、技术栈

#### 优先级 3：教育部及教育厅官网

- **教育部官网** (moe.gov.cn)

**职业教育数据源**：
