## 不同角色的关注点

### CEO/总经理

**核心关注**：
1. **战略价值**：如何支撑企业数字化转型战略
2. **ROI**：投资回报率、成本效益
3. **风险控制**：数据安全、合规风险
4. **竞争优势**：如何帮助企业在竞争中获胜

**沟通策略**：
- 强调战略价值，而非产品功能
- 量化 ROI，用数据说话
- 强调风险控制，满足合规要求
- 提供行业标杆案例

**话术模板**：
- "XX企业用这套方案让全公司效率提升30%，一年省XX万"
- "数字化转型只做了一半，数据安全出问题会被问责"
- "打造成行业数字化标杆"

---

### CIO/CTO

**核心关注**：
1. **技术架构**：如何与现有系统集成
2. **运维成本**：部署、维护、升级成本
3. **安全性**：数据安全、权限管理
4. **可扩展性**：未来扩展能力

**沟通策略**：
- 强调技术架构的开放性
- 强调运维成本的优势
- 强调安全能力和认证
- 强调可扩展性和未来发展

**话术模板**：
- "一套平台管到底，运维成本降一半"
- "系统越上越多，数据孤岛、安全漏洞都管不过来"
- "建设企业级数字化底座"
- "API 丰富，易于与现有系统集成"

---

### 业务负责人（部门经理）

**核心关注**：
1. **业务效率**：如何提升团队效率
2. **协作体验**：是否易用、是否流畅
3. **业务价值**：能否解决实际业务问题
4. **学习成本**：团队学习成本高低

**沟通策略**：
- 强调业务效率的提升
- 强调产品的易用性
- 强调解决实际业务痛点
- 强调学习成本低

**话术模板**：
- "不用再为文档打架，业务响应快一倍"
- "跨部门协作太难，一个文档反复改版本"
- "打造数字化协同团队"
- "界面熟悉，学习成本低"

---

### IT 经理/技术负责人

**核心关注**：
1. **技术实现**：如何部署、如何配置
2. **兼容性**：与现有系统的兼容性
3. **权限管理**：权限精细度、管理能力
4. **数据迁移**：数据迁移难度、迁移成本

**沟通策略**：
- 提供详细的技术方案
- 强调兼容性和集成能力
- 强调权限管理能力
- 提供数据迁移方案

**话术模板**：
- "支持多种部署方式，灵活适配"
- "API 丰富，易于与现有系统集成"
- "七重权限管理，精细化控制"
- "提供专业数据迁移服务"

---

### 采购负责人

**核心关注**：
1. **价格**：采购成本、预算范围
2. **性价比**：功能与价格的匹配度
3. **服务**：售后服务、技术支持
4. **合同条款**：付款方式、违约责任

**沟通策略**：
- 强调性价比优势
- 强调 TCO（总拥有成本）而非单价
- 强调服务质量和响应速度
- 灵活的合同条款

**话术模板**：
- "一套方案覆盖协同+会议+云盘，TCO 更优"
- "本土化服务，响应更快"
- "灵活的付款方式，降低资金压力"

## 决策链条分析

### 典型决策链

**大型企业（国企/央企）**：

```
业务部门（需求发起）→ IT 部门（技术评估）→ 采购部门（商务谈判）→ 法务部门（合同审核）→ 分管领导（审批）→ 一把手（最终决策）
```

**关键人物**：
- **决策者**：一把手（CEO/董事长）、分管领导
- **评估者**：CIO/CTO、IT 经理
- **影响者**：业务部门负责人、法务负责人
- **使用者**：业务部门员工

---

**中小企业**：

```
业务部门（需求发起）→ 老板（直接决策）
```

**关键人物**：
- **决策者**：老板/CEO
- **影响者**：业务负责人
- **使用者**：员工

---

**金融机构**：

```
业务部门（需求发起）→ IT 部门（技术评估）→ 风控/合规部门（风险评估）→ 采购部门（商务谈判）→ 分管领导（审批）→ 一把手（最终决策）
```

**关键人物**：
- **决策者**：一把手（行长/董事长）、分管领导
- **评估者**：CIO/CTO、风控负责人、合规负责人
- **影响者**：业务部门负责人、采购负责人
- **使用者**：员工

## 沟通策略建议

### 自上而下策略

**适用场景**：大型企业、国企/央企

**策略**：
1. 先搞定高层（CEO/分管领导），获得战略支持
2. 再配合 IT 部门进行技术评估
3. 最后支持业务部门进行试用

**优势**：
- 自上而下推动，阻力小
- 容易获得资源支持
- 决策效率高

**风险**：
- 高层决策可能因不了解实际需求而偏差
- 需要强大的高层关系

---

### 自下而上策略

**适用场景**：中小企业、无高层关系

**策略**：
1. 先支持业务部门试用，获得内部支持者
2. 通过业务部门推动 IT 部门评估
3. 最后由高层决策

**优势**：
- 需求明确，试用反馈真实
- 建立广泛的内部支持
- 风险较低

**风险**：
- 决策周期长
- 容易在 IT 评估环节被卡

---

### 多点突破策略

**适用场景**：大型企业、决策链复杂

**策略**：
1. 同时接触高层、IT 部门、业务部门
2. 针对不同角色提供不同价值主张
3. 形成内部支持网络

**优势**：
- 风险分散
- 形成合力
- 提高成功率

**风险**：
- 需要资源投入大
- 需要团队配合

## 高层四大关注点详解

本部分为高层拜访故事线技能（presale-executive-storyline）提供详细的关注点分析。

### 影响力

**核心诉求**：
- 个人政绩、打造标杆、行业领先
- 获得上级认可、行业影响力
- 政绩考核、晋升资本

**WPS 365 对接话术**：
- "打造成行业数字化标杆"
- "成为XX部委推广的典型案例"
- "某央企已经验证，走在行业前列"
- "获得国资委/工信部/银行业协会推广"

**适用角色**：
- 国企/央企 CEO
- 政府/事业单位负责人
- 追求行业影响力的民营企业 CEO

**典型场景**：
- 国企改革、数字化转型项目
- 政府数字化建设
- 行业标杆打造

---

### 业务增长

**核心诉求**：
- 核心业务指标增长
- 资产规模扩张
- 业务创新突破
- 营收增长、利润提升

**WPS 365 对接话术**：
- "效率提升40% → 业务响应快 → 收入增长"
- "支撑核心业务指标增长XX%"
- "数字化手段驱动业务创新"
- "项目交付周期缩短30% → 承接更多项目"

**适用角色**：
- 民营企业 CEO
- 业务负责人
- 追求业务增长的高层

**典型场景**：
- 业务扩张期
- 市场竞争激烈
- 需要快速响应市场

---

### 安全

**核心诉求**：
- 政策要求、监管合规
- 数据安全、风险控制
- 信创国产化
- 避免问责、零安全事故

**WPS 365 对接话术**：
- "等保三级、涉密认证"
- "数据不出境、自主可控"
- "满足信创国产化政策要求"
- "审计留痕，操作可追溯"
- "零安全事故，避免问责"

**适用角色**：
- 金融行业 CEO/CIO
- 政府/涉密单位负责人
- 国企/央企 CEO

**典型场景**：
- 信创改造项目
- 金融合规项目
- 政府涉密项目
