## 数据获取优先级

### 优先级1：用户上传文件（最优先）

**场景**：用户上传了客户资料、招标文件、内部文档

**优势**：
- 信息最准确、最详细
- 包含内部信息和数据
- 可直接提取关键信息

**使用方法**：
1. 使用 Read 工具读取文件
2. 提取关键信息：公司规模、业务结构、痛点、预算等
3. 标注数据来源为"用户上传文件"

**示例**：
```markdown
## 数据来源说明

### 一手数据来源
- 用户上传文件：客户内部需求文档.pdf（2025-01-15）
```

### 优先级2：巨潮网公告（权威来源）

**场景**：上市公司年报、半年度报告、重大事项公告

**优势**：
- 权威、准确、全面
- 包含业务规模、战略方向、IT投入等关键信息
- 免费公开

**URL格式**：
- 主站：http://www.cninfo.com.cn/new/index
- 搜索：http://www.cninfo.com.cn/new/search/keyword?searchkey=公司全称

**关键信息获取**：
1. **年度报告**：
   - 业务规模（营收、资产、用户数）
   - 战略方向和重点
   - IT投入和技术投入
   - 组织架构
   - 风险和挑战

2. **半年度报告**：
   - 最新业务进展
   - 中期业绩
   - 上半年重点工作

3. **重大事项公告**：
   - IT项目公告
   - 数字化转型项目
   - 重大项目采购

**使用方法**：
1. 使用 WebReader 工具抓取巨潮网页面
2. 配置参数：
   ```json
   {
     "url": "巨潮网URL",
     "return_format": "markdown",
     "retain_images": false,
     "timeout": 30,
     "with_links_summary": true
   }
   ```
3. 从返回的markdown内容中提取关键信息

**示例**：
```markdown
## 数据来源说明

### 一手数据来源
- 巨潮网：招商银行2024年年度报告 http://www.cninfo.com.cn/new/disclosure/stock?stockcode=600036&orgId=9900006613
```

### 优先级3：公司官网/官方渠道

**场景**：了解业务介绍、组织架构、新闻动态

**优势**：
- 官方权威信息
- 了解最新动态
- 了解产品和服务

**关键页面**：
1. **关于我们**：公司介绍、发展历程、企业文化
2. **投资者关系**：财报、公告、投资者演示
3. **业务介绍**：主要业务板块、产品和服务
4. **新闻中心**：最新动态、重大新闻
5. **招聘页面**：了解IT团队规模、技术栈

**使用方法**：
1. 使用 WebReader 工具抓取官网页面
2. 从返回内容中提取关键信息

**示例**：
```markdown
## 数据来源说明

### 一手数据来源
- 公司官网：招商银行官网 https://www.cmbchina.com
- 投资者关系：招商银行投资者关系 https://www.cmbchina.com/IR/
```

### 优先级5：WebSearch 最新信息（补充验证）

**场景**：搜索最新动态、补充验证前面获取的信息

**搜索关键词**：
- `公司名称 + 数字化转型`
- `公司名称 + 办公系统`
- `公司名称 + 协同办公`
- `公司名称 + IT采购`
- `公司名称 + 信创`
- `公司名称 + WPS`

**使用方法**：
1. 使用 WebSearch 工具搜索
2. 从搜索结果中提取最新动态
3. 补充验证前面获取的信息

**示例**：
```markdown
## 数据来源说明

### 二手数据来源
- WebSearch：招商银行数字化转型最新动态（2025-01-20搜索）
```

### 优先级6：实时信息查询（可选触发）

**触发条件**：
- 满足以下任一条件即触发实时信息查询
- 招投标信息：用户明确要求、research_depth为comprehensive、IT投入>1亿/年、include_bidding参数为true
- 子公司信息：用户明确要求、企业性质为集团、公司规模>1000亿、include_subsidiaries参数为true

详细使用方法请参考：[实时数据源使用指南](real-time-data-sources.md)

## 一手数据来源

### 巨潮网（cninfo）

**网址**：http://www.cninfo.com.cn

**适用对象**：所有上市公司

**关键信息**：
- 年度报告：业务规模、战略、IT投入
- 半年度报告：最新进展
- 重大事项公告：IT项目、采购信息

**使用技巧**：
1. 使用公司全称搜索，结果更准确
2. 优先查看年度报告"管理层讨论与分析"部分
3. 关注"核心竞争力"和"未来展望"部分
4. 查看最近3年的报告，了解发展趋势

### 公司官网

**关键页面**：
1. 首页：了解公司定位和品牌形象
2. 关于我们：公司介绍、发展历程
3. 投资者关系：财报、公告、演示材料
4. 业务介绍：主要业务和产品
5. 新闻中心：最新动态

**使用技巧**：
1. 投资者关系页面通常有最丰富的信息
2. 下载投资者演示材料（Presentation），信息密度高
3. 查看最近一年的新闻，了解最新动态

### 招聘网站

**推荐网站**：
1. 猎聘：https://www.liepin.com
2. 智联招聘：https://www.zhaopin.com
3. BOSS直聘：https://www.zhipin.com

**关键信息**：
- IT团队规模（根据招聘岗位数量估算）
- 技术栈（根据职位描述）
- IT预算（根据薪资水平估算）

**搜索关键词**：
- `公司名称 + 招聘`
- `公司名称 + IT`
- `公司名称 + 开发`
- `公司名称 + 产品经理`

### LinkedIn（领英）

**网址**：https://www.linkedin.com

**关键信息**：
- 关键人物背景（教育、经历）
- 组织架构
- 团队规模

**使用技巧**：
1. 搜索公司名称，查看公司页面
2. 查看关键人物的LinkedIn档案
3. 分析组织架构和团队规模

## WebReader使用指南

### 基本配置

```json
{
  "url": "目标URL",
  "return_format": "markdown",
  "retain_images": false,
  "timeout": 30,
  "with_links_summary": true,
  "with_images_summary": false
}
```

### 参数说明

- **url**：目标网页URL（必需）
- **return_format**：返回格式，推荐"markdown"
- **retain_images**：是否保留图片，推荐false（节省token）
- **timeout**：超时时间（秒），推荐30
- **with_links_summary**：是否包含链接摘要，推荐true
- **with_images_summary**：是否包含图片摘要，推荐false

### 使用示例

#### 示例1：抓取巨潮网年报

```json
{
  "url": "http://www.cninfo.com.cn/new/disclosure/stock?stockcode=600036&orgId=9900006613",
  "return_format": "markdown",
  "retain_images": false,
  "timeout": 30,
  "with_links_summary": true
}
```

#### 示例2：抓取公司官网

```json
{
  "url": "https://www.cmbchina.com/About/",
  "return_format": "markdown",
  "retain_images": false,
  "timeout": 30,
  "with_links_summary": true
}
```

### 最佳实践

1. **优先抓取关键页面**���
   - 年度报告（巨潮网）
   - 投资者关系（官网）
   - 业务介绍（官网）

2. **注意页面大小**：
   - 年度报告通常很大，可能需要分段读取
   - 如果页面太大，尝试抓取特定章节

3. **验证数据准确性**：
   - 交叉验证多个来源
   - 优先使用官方数据
   - 标注数据获取日期

4. **保存数据来源**：
   - 记录每个数据的来源URL
   - 记录数据获取日期
   - 标注数据时效性

## WebSearch使用指南

### 搜索策略

#### 策略1：组合搜索

**搜索公式**：`公司名称 + 关键词`

**关键词推荐**：
- 数字化转型
- 协同办公
- IT采购
- 信创
- 国产化
- 办公系统
- WPS
- 金山办公

**示例**：
- `招商银行 数字化转型`
- `招商银行 协同办公`
- `招商银行 信创`

#### 策略2：问题搜索

**搜索公式**：`公司名称 + 问题`

**示例**：
- `招商银行 协作办公痛点`
- `招商银行 IT系统问题`
- `招商银行 数字化转型挑战`

#### 策略3：竞品搜索

**搜索公式**：`公司名称 + 竞品名称`

**示例**：
- `招商银行 钉钉`
- `招商银行 飞书`
- `招商银行 企业微信`

### 最佳实践

1. **使用多个关键词**：
   - 不要只搜索一个关键词
   - 组合使用多个关键词
   - 验证多个搜索结果

2. **关注最新信息**：
   - 优先查看最近1年的结果
   - 注意信息的时效性
   - 标注信息发布日期

3. **交叉验证**：
   - 不要只依赖一个搜索结果
   - 验证多个来源的信息
   - 注意信息的一致性

4. **记录来源**：
   - 记录每个信息的来源URL
   - 记录搜索日期
   - 标注信息发布日期

### 数据标注规范

#### 标注格式

```markdown
**数据来源**：[来源名称] [URL] [获取日期]
**数据时效性**：[数据发布日期]
**数据可靠性**：[高/中/低]
```

#### 示例

```markdown
| 字段 | 内容 | 数据来源 |
|------|------|----------|
| 公司全称 | 招商银行股份有限公司 | 巨潮网2024年报 http://www.cninfo.com.cn |
| 业务规模 | 营收3311亿元 | 巨潮网2024年报（2024-03-26发布） |
| 数字化进展 | 移动端交易占比85% | 官网（2025-01-15访问） |
```

### 数据验证方法

1. **交叉验证**：
   - 同一信息，验证多个来源
   - 不一致时，优先使用官方来源

2. **时间验证**：
   - 检查数据的发布时间
   - 优先使用最新数据

3. **逻辑验证**：
   - 检查数据是否合理
   - 注意明显的错误或异常值

4. **完整性验证**：
   - 检查关键字段是否完整
   - 不完整时，标注"待确认"

## 数据使用建议

### 数据收集建议

1. **按优先级收集**：
   - 优先使用用户上传文件
   - 其次使用巨潮网年报
   - 再次使用官网和行业协会
   - 最后使用WebSearch补充

2. **多源验证**：
   - 关键信息需要多源验证
   - 不一致时，以官方来源为准

3. **时效性优先**：
   - 优先使用最新数据
   - 标注数据发布时间

### 数据使用建议

1. **标注来源**：
   - 所有数据必须标注来源
   - 包括URL和获取日期

2. **评估质量**：
   - 评估数据准确性
   - 评估数据时效性
   - 评估数据完整性

3. **明确限制**：
   - 明确数据的局限性
   - 不确定的数据，标注"待确认"
   - 避免过度推断

### 报告撰写建议

1. **数据来源说明**：
   - 专门章节说明数据来源
   - 列出所有一手数据来源
   - 列出所有二手数据来源

2. **数据时效性说明**：
   - 说明数据的时间范围
   - 说明数据的更新频率
   - 说明数据的局限性

3. **数据可靠性说明**：
   - 说明数据的可靠性
   - 说明数据的验证方法
   - 说明数据的不确定性
