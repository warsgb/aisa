# 电梯演讲

## 使用方式

本技能根据您选择的客户和目标角色，自动生成 30 秒电梯演讲话术。

### 参数说明

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| **customer_name** | 文本 | 是 | 客户公司名称，自动填充当前选择的客户 |
| **industry** | 文本 | 否 | 客户所属行业，自动从客户基础信息填充，也可手动输入 |
| **target_role** | 文本 | 是 | 目标角色（如：CEO/总经理、CIO/CTO、业务负责人、IT经理）|
| **hook_type** | 下拉选择 | 否 | 钩子类型（焦虑版/贪婪版/愿景版/全部），默认生成全部 |
| **include_pain_point** | 复选框 | 否 | 是否融入行业痛点分析，默认开启 |

### 使用示例

1. **快速生成**：选择客户后，系统自动填充公司和行业，您只需选择目标角色即可生成
2. **精准定制**：选择特定钩子和行业痛点，生成更精准的话术
3. **批量生成**：选择"全部"钩子类型，一次性获得三套话术版本

---

#### 2. 贪婪版（Greed）

**核心逻辑**：激发贪婪，强调行动的收益

**特点**：
- 80-100字（2-3句话）
- 量化收益
- 强调快速见效

**适用场景**：
- 业务负责人
- 效率导向型客户
- 需要快速证明价值

**模板**：
- "[角色称谓]，一套方案让[量化收益]。某[同类型客户]用我们的方案[量化成果]。想了解怎么做吗？"

**示例**：
- "张总，一套方案让全公司效率提升30%，一年省500万。某银行用我们的方案减少了30%的沟通成本，IT运维成本降了一半。想了解怎么做吗？"

---

#### 3. 愿景版（Vision）

**核心逻辑**：描绘愿景，激发想象

**特点**：
- 100-120字（3-4句话）
- 描绘未来场景
- 强调长期价值

**适用场景**：
- 一把手（CEO/董事长）
- 战略导向型客户
- 需要建立长期合作

**模板**：
- "[角色称谓]，打造[愿景目标]。我们的[解决方案]可以帮助[实现路径]。某[同类型客户]已经[标杆案例]，我们也能帮您做到。"

**示例**：
- "张总，打造成行业数字化标杆。我们的协同办公平台可以帮助建设企业级数字化底座，支撑未来5-10年发展。某银行已经用我们的方案实现了全行协同效率提升30%，我们也能帮您做到。"

### 第四步：应用金字塔原理

**金字塔结构**：

1. **结论先行**：一句话核心观点
2. **论据支撑**：2-3个支撑点
3. **行动号召**：明确的下一步

**示例**：

**结论先行**：
- "一套方案让全公司效率提升30%，一年省500万。"

**论据支撑**：
- "某银行用我们的方案减少了30%的沟通成本，IT运维成本降了一半。"
- "我们的平台覆盖协同+会议+云盘，一体化方案降低 TCO。"

**行动号召**：
- "想了解怎么做吗？我们可以约个时间详细聊。"

#### 金融行业-银行

**典型痛点**（来自框架）：
- 系统割裂：OA、文档、会议、消息等工具独立运行
- 多端切换：多应用消息割裂，需多端切换处理
- 信息过载：跨部门消息刷屏导致关键信息遗漏

**CEO/总经理 - 焦虑版**：
- "张总，金融行业数据安全是红线，出了问题会被监管问责。我们的私有化部署可以保证数据不出境，满足等保三级和涉密认证，某银行已经在用了。"

**CIO/CTO - 贪婪版**：

| 制造业 | "供应链协同效率低" → "一个变更通知不到所有人" |

**痛点融入话术示例**：

原有话术：
