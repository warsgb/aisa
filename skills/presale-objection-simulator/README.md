# WPS 365 异议攻防模拟器

通过 AI 角色扮演挑剔客户，提供沉浸式异议应对训练。帮助售前人员在模拟场中"死"几次，真正拜访时心如止水。

---

## 核心价值

- **���浸式角色扮演**：AI 扮演各种类型的挑剔客户
- **多轮攻防对话**：不轻易接受解释，持续追问
- **教练复盘模式**：暂停时提供专业点评和优化建议
- **心态脱敏训练**：在模拟场中"死"，实战时才能活

---

## 客户角色类型

| 角色类型 | 核心特征 | 攻击风格 |
|---------|---------|---------|
| **技术控CIO** | 多疑、技术细节敏感、反感SaaS | 技术质疑、刨根问底 |
| **务实CEO** | 关注ROI、时间紧张 | 直接质疑、不耐烦 |
| **合规狂人** | 风险厌恶、审计恐惧 | 极端假设、风险放大 |
| **价格敏感型** | 成本导向、比价专家 | 对比竞品、压价 |
| **信创强硬派** | 政策驱动、国产化执念 | 质疑资质、政策施压 |

---

## 快速开始

### 基础用法

```
用户：帮我模拟一个技术控CIO，我在推销混合云方案
```

系统将启动模拟，AI 扮演技术控CIO 与你进行攻防对话。

### 完整参数

```
用户：帮我模拟一个信创强硬派CEO，性格是多疑和政策敏感，
我在推销私有化部署方案，攻击强度设为harsh，关注数据安全和信创认证
```

---

## 对话控制命令

在模拟过程中，可以使用以下命令：

| 命令 | 功能 |
|------|------|
| `暂停` | 暂停对话，进入教练复盘模式 |
| `继续` | 结束复盘，继续对话 |
| `换人` | 更换客户角色 |
| `更狠` | 增加攻击强度 |
| `温柔` | 降低攻击强度 |

---

## 攻击强度说明

| 强度 | 描述 | 典型表现 |
|------|------|---------|
| `mild` | 温和 | 友好提问，给台阶下 |
| `medium` | 中等 | 直接质疑，偶尔追问 |
| `harsh` | 严厉 | 尖锐问题，持续追问 |
| `brutal` | 苛刻 | 极端质疑，打断施压 |

## 与其他 Skills 的组合使用

### 前置组合：战略解码 → 异议模拟

先用 `presale-strategy-decoder` 分析客户，了解推测痛点后，再进行针对性的异议模拟。

### 后置组合：异议模拟 → 高层故事线

通过异议模拟暴露应对不充分的异议，然后用 `presale-executive-storyline` 优化故事线论据。

### 并行组合：需求挖掘 + 异议模拟

用 `presale-requirements-discovery` 挖掘需求，同时用异议模拟训练应对能力。

---

## 常见异议类型

| 异议类别 | 典型问题 | 应对要点 |
|---------|---------|---------|
| **数据安全** | "数据到底在哪？会不会出境？" | 私有化部署、等保三级 |
| **竞品对比** | "比微软好在哪里？" | 信创支持、私有化 |
| **成本质疑** | "价格太高，预算不够" | TCO更优、对比M365降低70% |
| **技术怀疑** | "架构不合理，有单点故障" | 自研引擎"轻舟" |
| **合规风险** | "信创认证有吗？" | 三大架构认证 |
| **实施风险** | "实施周期多长？失败怎么办？" | 分阶段实施、试点方案 |
