# WPS 365 财报年报战略解码技能

从客户年报、战略规划文件中解码销售机会，自动识别战略关键词，深度推理业务痛点，精准匹配 WPS 365 解决方案。

## 触发条件

当用户提供以下任一信息时，启动本技能：

1. **企业年报**：上市公司年报、社会责任报告、ESG 报告
2. **战略规划**：战略规划、三年规划、数字化转型规划
3. **官网信息**：公司官网的战略介绍、业务介绍
4. **公司名称**：仅提供公司名称，自动搜索年报信息

**示例**：
- "帮我分析招商银行2025年年报"
- "解码国家电网的战略规划"
- "从中国平安的年报中找销售机会"

## 使用流程

### ⚠️ 重要：年份使用规则

**当前年份规则**：
- 当前系统年份：**2026年**
- 搜索年报时优先搜索**2025年年报**
- 搜索关键词格式：`公司名称 + 2025 + 年报`
- 报告中所有日期标注使用当前年份

### 第一步：获取文档内容

#### 优先级1：用户上传文件（最优先）

如果用户上传了年报文件或战略文档：
- 使用 Read 工具读取文件内容
- 提取关键信息：战略方向、业务挑战、IT投入、数字化进展

#### 优先级2：权威数据源（金融/国企客户）

**金融行业客户（巨潮网）**：
- URL 格式：`http://www.cninfo.com.cn/new/search/keyword?searchkey=公司全称`
- 搜索关键词：`公司名称 + 2025 + 年度报告`
- 使用 WebReader 工具获取年报内容

**国企客户（官网/国资委）**：
- 公司官网：投资者关系板块
- 国资委官网：搜索公司名称
- 使用 WebSearch + WebReader 获取信息

#### 优先级3：官网战略信息

- 公司官网：战略介绍、业务介绍、新闻动态
- 使用 WebReader 工具获取

#### 优先级4：WebSearch 补充

- 搜索组合：`公司名称 + 2025 + 年报`
- 搜索组合：`公司名称 + 数字化转型 + 2025`
- 搜索组合：`公司名称 + 战略 + 2025`

### 第二步：关键词提取与分类

从文档中提取战略关键词，按以下四类分类：

#### 1. 降本增效类

**关键词库**：
- 集约化、扁平化、流程优化、协同提效、降本增效
- 数字化转型、精益管理、效率提升、成本控制
- 优化、重构、整合、共享、协同

**映射业务场景**：
- 流程优化 → 跨部门协作需求 → WPS 365 协同文档
- 协同提效 → 团队沟通需求 → 金山协作 IM
- 成本控制 → IT成本压力 → 一体化平台降低 TCO

#### 2. 合规风控类

**关键词库**：
- 内控、审计、留痕、权限管理、数据治理
- 合规、风险控制、监管报送、信息安全
- 审计追溯、操作留痕、权限精细

**映射业务场景**：
- 审计留痕 → 审计要求 → WPS 365 审计留痕功能
- 权限管理 → 数据安全需求 → 七重权限管理
- 合规要求 → 国产化需求 → 信创全栈支持

#### 3. 数字创新类

**关键词库**：
- 数字化、智能化、移动化、云化
- 数字化转型、智慧XX、AI赋能
- 云平台、移动办公、智能协作

**映射业务场景**：
- 数字化转型 → 协同办公需求 → WPS 365 一体化平台
- 智能化 → AI 需求 → WPS AI 企业版
- 移动化 → 移动办公需求 → WPS 365 移动端

#### 4. 安全可信类

**关键词库**：
- 信创、国产化、自主可控、数据安全
- 等保、涉密、合规、私有化
- 数据不出境、安全可控

**映射业务场景**：
- 信创/国产化 → 信创改造需求 → 信创全栈支持
- 数据安全 → 私有化需求 → 私有化部署
- 等保/涉密 → 合规认证需求 → 等保三级/涉密认证

### 第三步：痛点深度推理

基于关键词深度推理业务痛点：

**推理链条**：关键词 → 业务场景 → 隐性阻碍 → 推测痛点

**示例推理**：

**关键词**："数字化转型"

**业务场景**：企业在推进数字化转型，但可能面临：
- 系统越上越多，数据孤岛严重
- 跨部门协作困难，流程不顺畅
- 文档管理混乱，版本不可追溯

**隐性阻碍**：
- IT部门疲于应付系统维护
- 业务部门协作效率低
- 管理层看不到数字化转型的实际效果

**推测痛点**：
- "系统分散，数据孤岛，协作效率低"
- "文档版本混乱，审计追溯困难"
- "IT运维成本高，但业务满意度低"

**WPS 365 解决方案**：
- 统一平台 → 打破数据孤岛
- 协同文档 + 版本管理 → 版本可追溯
- 一体化平台 → 降低运维成本

- **破冰话术**：用于初次沟通的话术（如"张总，我们专注为企业提供协同办公解决方案，帮助类似企业效率提升了30%"）

### 第五步：生成三种话术

针对不同场景生成话术：

#### 1. 初次见面话术

**特点**：短、吸引人、建立信任

**模板**：
- "[对方称谓]，我们专注为企业提供[核心价值]，帮助[同类型客户][量化成果]。不知道贵公司在[相关领域]是否有类似需求？"

**示例**：
- "张总，我们专注为企业提供协同办公解决方案，帮助招商银行类似企业效率提升了30%。不知道贵公司在跨部门协作方面是否有类似需求？"

#### 2. 方案汇报话术

**特点**：针对性强、解决痛点、量化价值

**模板**：
- "基于您在[场合]提到的[痛点]，我们的[解决方案]可以实现[核心能力]。某[同类型客户]用这个方案[量化成果]。"

**示例**：
- "基于您在年报中提到的'数字化转型'，我们的协同办公平台可以实现统一协作。某银行用这个方案减少了30%的沟通成本，IT运维成本降低了一半。"

#### 3. 投标答辩话术

**特点**：差异化优势、合规安全、信心十足

**模板**：
- "相比竞品，我们有[数字]点核心优势：一是[优势1]，二是[优势2]，三是[优势3]。这些优势在[同类型客户]已经得到验证。"

**示例**：
- "相比竞品，我们有三点核心优势：一是信创全栈支持，满足国产化替代要求；二是私有化部署能力，保证数据不出境；三是等保三级认证，满足金融行业合规要求。这些优势在某银行已经得到验证。"
